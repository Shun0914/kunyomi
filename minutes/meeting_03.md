# 第3回ミーティング議事録

## チーム運営と役割分担

### ファシリテーション体制の見直し

現在しゅんすけがミーティングの進行を担当している状況について議論し、以下の方針で合意：

* 役割の固定化を避け、得意分野を活かしながらも柔軟な分担を目指す  
* 各メンバーが積極的に参加し、特定の人に負担が集中しないよう配慮する

### 役割の明確化

| 役割 | 担当 | 責任範囲 |
| ----- | ----- | ----- |
| **PdM（プロダクトマネージャー）** | よこち | プロダクトの方向性決定、全体設計、問いの設定 |
| **UXデザイナー** | なおみん | UX部分のリード、具体的なワークフレームの進行 |
| **PM（プロジェクトマネージャー）** | （未確定） | 決定事項のデリバリー、進捗管理 |

## 年内スケジュールと作業計画

### 確定済みミーティング日程

| 日程 | 目的 |
| ----- | ----- |
| **12月21日** | リーンキャンバスのコア部分（問題・解決策・価値）を決定 |
| **12月22日** | 役割分担と具体的なタスク決定 |
| **12月30日** | 年内作業の完了と最終確認 |

### 作業の優先順位

**最重要：** リーンキャンバスの問題・解決策・価値の3要素を確定

**優先度低：** その他の要素（コスト、チャネル、メトリクス等）は役割分担後に個別作業

**年内目標：** 要件定義関連の課題を完了させる

---

## 具体的な進め方

### 12月21日までの準備作業

1. **よこちの作業：** ペインポイントを3つ程度リストアップ  
2. **全員の作業：** それらの解決策と価値提案を検討  
3. **21日の議論：** 複数のアプローチを持ち寄り、チームとして最適な方向性を決定

### 課題の取り組み方針

* 全ての課題を完了することよりも、**プロダクト実現に必要な要素を優先**  
* データベース設計やプロトタイプ作成は年明け以降に延期  
* 競合調査等は必要に応じて実施し、結果を共有
